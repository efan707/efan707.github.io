web设计之图片格式

日期: 2015-04-16   |   分类: CSS   |  

web设计时，会遇到图片格式的选择问题，现如今相关的图片格式有：GIF、PNG、JPEG、APNG、WEBP等。

GIF

图像互换格式（GIF，Graphics Interchange Format）是一种位图图形文件格式，以8位色（即256种颜色）重现真彩色的图像。它采用LZW压缩算法进行编码，有效地减少了图像文件在网络上传输的时间，它是目前广泛应用于网络传输的图像格式之一。
GIF自1987年由CompuServe公司引入后，因其体积小而成像相对清晰，特别适合于初期慢速的互联网，而从此大受欢迎。它采用无损压缩技术，只要图像不多于256色，则可既减少文件的大小，又保持成像的质量。
对于超过256种颜色的图像，如果保存为GIF格式，必然会导致某些信息的丢失，最终变为了有损压缩。
GIF只支持1位的透明度，要么完全透明，要么完全不透明。但是它支持基于帧的简单动画，可用PS、FLASH以及GIF专业工具进行制作。

PNG

便携式网络图形（Portable Network Graphics，PNG）是一种无损压缩的位图图形格式，支持索引、灰度、RGB三种颜色方案以及Alpha通道等特性。PNG的开发目标是改善并取代GIF作为适合网络传输的格式而不需专利许可，所以被广泛应用于互联网及其他方面上。
PNG提供了两种色深模式：24位和8位。24位色很大，但却保证了完全无损。——《CSS禅意花园》
PNG还支持两种透明度设置：1bit或8位。前者跟GIF一样，要么全透，要么不透；后者支持256级的透明度。
IE6中不支持PNG的透明，需要使用滤镜等方式去解决。

JPEG

JPG全名是JPEG。JPEG图片以 24 位颜色存储单个位图。JPEG 是与平台无关的格式，支持最高级别的压缩，不过，这种压缩是有损耗的。JPEG不支持任何程度的透明度。

APNG

APNG（Animated Portable Network Graphics）格式是PNG的位图动画扩展，但未获PNG组织官方认可。它的扩展方法类似GIF 89a，仍对原版PNG保持向下兼容，由于APNG是PNG的扩展，所以它支持半透明，自然就不会有锯齿的问题。APNG第1帧为标准PNG图像，剩余的动画和帧速等数据放在PNG扩展数据块里，因此只支持原版PNG的软件会正确显示第1帧。

WebP

WebP（发音 weppy， 项目主页 ），是一种支持有损压缩和无损压缩的图片文件格式，派生自图像编码格式 VP8。根据 Google 的测试，无损压缩后的WebP 比 PNG 文件少了 45％ 的文件大小，即使这些 PNG 文件经过其他压缩工具压缩之后，WebP 还是可以减少 28％ 的文件大小。
WebP 的优势体现在它具有更优的图像数据压缩算法，能带来更小的图片体积，而且拥有肉眼识别无差异的图像质量；同时具备了无损和有损的压缩模式、Alpha 透明以及动画的特性，在 JPEG 和 PNG 上的转化效果都非常优秀、稳定和统一。
