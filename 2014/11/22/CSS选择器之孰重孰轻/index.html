CSS选择器优先级判定

發表於 2014-11-22   |   分類於 CSS

CSS不同方式引入

CSS优先级即CSS样式在浏览器中被解析出来的先后顺序。
对于不同方式引入的样式，一般情况下，优先级如下：
外部样式(External style sheet) < 内部样式(Internal style sheet) < 内联样式(Inline style)
——如果外部样式放在内部样式的后面，则外部样式将覆盖内部样式；JS设置的样式为内联样式。
总之一句话：

样式位置愈近之，优先级愈高之

选择器权重

约定俗成，我们用(A,B,C,D)来表示CSS选择器的权重，其中：
A表示CSS是否为内联样式，是为1，否为0
B表示ID选择器的个数
C表示类、伪类和属性选择器的个数
D表示标签、伪元素的个数（例如，p:first-letter 这样的选择器，计算为0, 0, 0, 2）。
具体如下表所示：

选择器类型 权重(A,B,C,D) 例子
内联样式 (1,0,0,0)
ID选择器 (0,1,0,0) #IDName{}
CLASS类选择器 (0,0,1,0) .className{}
伪类选择器 (0,0,1,0) :hover{}
属性选择器 (0,0,1,0) [href]{}
标签选择器 (0,0,0,1) p{}

优先级比较规则：从权重左边的第一位开始比较大小，如果较大，则优先级较高；反之，则优先级较低；如果一样，继续往右逐位比较。(IE不支持!important)

权重优先级：important > 内联 > ID > 类 | 伪类 | 属性选择 > 标签 | 伪对象 > 继承 > 通配符

特殊情况

上面的优先级适用于一般场合，但对于一些极致的情况，则不再适应
——IE, Mozilla或WebKit浏览器(如FireFox)中，一个ID的权重为(0,1,0,0)，256个CLASS类选择器的权重为(0,0,256,0)，但他们的优先级是一样的；Opera、Chorme中，则符合优先级判比规则，权重为(0,1,0,0)的较高。可参照文章Extreme specificity overriding a CSS ID with 256 chained Classes。
具体原因为：

@ryanseddon From hg.mozilla.org/mozilla-centra… Gecko overflows the count of classes into the count of IDs, each of which holds 8 bits.
— Cameron McCormack (@heycam) August 16, 2012

@ryanseddon and to close the loop: yes, opera uses 16 bits instead of 8. bring on 65536 classes…
— patrick h. lauke (@patrick_h_lauke) August 16, 2012

IE, Mozilla或WebKit浏览器，他们的类名(classes)都是以8字节(256位)字符串存储的，当同时指定256个类时，溢出到id区域；Opera、Chorme浏览器class类名的存储是以16字节的字符串，想发生class溢出到id的话，需要连续65536个class。
