Cookie为何物

Cookie，有时也用其复数形式Cookies，指某些网站为了辨别用户身份、进行session跟踪而储存在用户本地终端上的数据（通常经过加密）。cookie机制将信息存储于用户硬盘，因此可以作为全局变量，这是它最大的一个优点。它可以用于保存用户登录状态、跟踪用户行为、定制页面等等。cookie的缺点主要集中于安全性和隐私保护。

操作Cookie

每个cookie都是一个名/值对，在cookie 的名或值中不能使用分号（;）、逗号（,）、等号（=）以及空格。如果有特殊符号，需要用escape()函数进行编码（它能将一些特殊符号使用十六进制表示，例如空格将会编码为“20%”），从而可以存储于cookie值中。编码的值需要使用unescape()进行解码。下面是对Cookie的主要操作：
1、添加Cookie
该函数可以接收6个参数：Cookie名称，Cookie值，过期时间expireDays，Cookie的路径path，Cookie关联域名domain，安全机制secure。
a、expireDays：为0时不设定过期时间，即当浏览器关闭时cookie自动消失。
b、路径path：在某个页面创建了一个cookie，那么该页面所在目录中的其他页面可以访问该cookie。同时，其所在目录的子目录也可以访问它。如在www.efan.me/a.html中所创建的cookie，可以被www.efan.me/b.html以及www.efan.me/sub/c.html所访问。如果要使cookie在整个网站下可用，可以将path指定为根目录，例如：

  if ( getCookie( name ) ) document.cookie = name + '=' +

cookie.js插件

jquery提供了一个操作Cookie的插件cookie.js，使用此插件，需前后引入query-2.1.1.min.js和jquery.cookie.js，操作Cookie命令如下：

$.cookie('name', 'value');  //添加Cookie
$.cookie('name', 'value', { expires: 7 });  //添加过期时间为7天的Cookie
$.cookie('name', 'value', { expires: 7, path: '/' });  //带过期时间和路径
$.cookie('name');     //read the cookie
$.cookie();           //Read all available cookies
$.removeCookie('name'); // 删除cookie
$.removeCookie('name', { path: '/' });    //删除指定路径的cookie

function addCookie( name, value, expireDays, path, domain, secure ) {
  var today = new Date();
  today.setTime( today.getTime() );
  if ( expireDays ) {
   expireDays = expireDays * 1000 * 60 * 60 * 24; 
  }
  var expires_date = new Date( today.getTime() + (expireDays) );
  document.cookie = name+'='+escape( value ) +
  ( (expireDays) ? ';expireDays='+expires_date.toGMTString() : '' ) + //expires.toGMTString()
  ( ( path ) ? ';path=' + path : '' ) +
  ( ( domain ) ? ';domain=' + domain : '' ) +
  ( ( secure ) ? ';secure' : '' );
} 

function getCookie( name ) {
  var start = document.cookie.indexOf( name + "=" ); 
  var len = start + name.length + 1; 
  if ( ( !start ) && ( name != document.cookie.substring( 0, name.length ) ) ) {
  return null;
  }
  if ( start == -1 ) return null; 
  var end = document.cookie.indexOf( ';', len ); 
  if ( end == -1 ) end = document.cookie.length; 
  return unescape( document.cookie.substring( len, end ) );
}

Gh-pages分支

日期: 2014-11-01   |   分类: Github   |  

前言

之前用Github-pages+Hexo搭建了博客，再绑定自己的域名，各种自在。近几日，闲来无事，想做一些web项目，但苦于没有很满意的平台。苦寻之，原来近在眼前——Github利用gh-pages分支可以很快创建静态项目主页。相比较什么Sae/Jae/Bae而言，它速度快、限制少，还可以关联自己的博客域名（绑定efan707.github.io的yichao.me），只能说没有更好。

Hello world

GitHub会为每个账号分配一个二级域名[user-id].github.io作为用户的Github-pages地址。其实，用户还可以利用gh-pages分支来创建项目主页，而项目主页的访问地址为：[user-id].github.io/[project-name]。如果博客绑定了独立域名，还可以通过——[yourDomainName]/[project-name]来访问。下面就以HelloWorld为例，讲解下gh-pages的用法。
1、打开Github，创建一名为HelloWorld的repository。
2、本地git克隆HelloWorld.git。

3、将静态网页文件(主页文件名为index.html)拷贝到HelloWorld文件夹下。
4、使用git命令创建分支gh-pages

5、切换分支到gb-pages并将HelloWorld文件夹中的内容（文件和文件夹）push到远程。

$ git checkout gh-pages
$ git add index.html
$ git commit -m "push index"
$ git push -u origin gh-pages

完成以上步骤后，便成功创建了一个简单的项目主页，地址为http://yichao.me/HelloWorld/
